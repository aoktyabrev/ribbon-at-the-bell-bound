# C2-R prereg DRAFT (не утверждён). Граница режима κ*: где ломается факторизация.
Мотив (C2-L): Факт 1 — κ→∞ даёт PR-box (CHSH=4); Факт 2 — измеренный режим
(κ_eff≈0.065, k_f×{1,4}) факторизуется (F_s<6.3·10⁻⁴). Между ними — граница
κ*(N,k_c). C2-R ищет излом, повышая k_f (⇒ κ_eff↑), и меряет прямую CHSH за ним.

## Стадия R0 — стабильность интегратора при больших k_f (ДО науки)
При больших k_f |∇E_twist| ∝ k_f ⇒ ланжевен-шаг lr·∇E растёт ⇒ неустойчивость
scan-carry (как fp-грабли цикла 1). Правило lr (предложение, на утверждение):
  lr = lr0 · min(1, 4/k_f_mult),  lr0=5e-3.
(k_f×8 ⇒ lr0/2; ×32 ⇒ lr0/8; ×128 ⇒ lr0/32; ×512 ⇒ lr0/128 — эффективный
шаг lr·k_f ограничен сверху.) Смоук M≤8: ТОЛЬКО rej_per_step и конечность
выходов (нет NaN/Inf, |u|=1), БЕЗ снятия наблюдаемых. KILL R0: rej_per_step
не стабилизируется (>0.01) или非конечности при выбранном lr ⇒ правило lr
пересмотреть до науки; при неудаче — неявный/адаптивный интегратор (решение
архитектора).

## Основная кампания (после R0-прохода)
F_s(k_f) при k_f×{8, 32, 128, 512}, N∈{8, 32} (κ-скейлинг длины), M=4800, T=0.
Протокол C2-F (сопряжённые прогоны, общий λ, a=ê фикс, b→b'(Δγ), считывание ê;
DEGENERATE колонкой, sign(0)→+1, зеркала). Первичный Δγ=π/2 (максимальный
рычаг); при F_s>0 в ячейке — доснять полную сетку Δγ∈{π/8,π/4,3π/8,π/2} И
прямые 4 CHSH-коррелятора ТАМ ЖЕ (стандартная геометрия a∈{0,π/2}, b∈{π/4,3π/4}).

## Гипотезы (прогнозы архитектора — в журнал до данных)
H-R1 (излом существует): F_s > 3σ_F при некотором k_f ≤ 512, N=32.
  KILL: F_s < 3σ_F на всей лестнице k_f до 512 (N=32) ⇒ факторизация
  устойчива до κ_eff(×512); граница за пределом окна — режим шире, чем
  ожидалось, PR-предел (Факт 1) достигается только в строгом κ→∞.
H-R2 (κ-скейлинг): излом при МЕНЬШЕМ k_f для N=8, чем для N=32 (короче лента ⇒
  выше эффективная жёсткость на длину). KILL: одинаковый порог по k_f для
  обоих N (в пределах шага лестницы) ⇒ κ* не масштабируется длиной как ожидалось.
H-R3 (S_direct): при F_s>0 прямая CHSH растёт от 2ρ (LHV) вверх.
  ЛЮБОЕ S>2 — СТОП-АУДИТ ДО интерпретации (Пирл/симметрия/no-signaling
  маргиналов проверяются ПЕРВЫМИ, стандарт цикла 1). Интерпретация «совместный
  режим / выход за LHV» ЛЕГАЛЬНА только после: (i) аудит пройден, (ii) F_s>0 в
  той же ячейке, (iii) маргиналы P(s|a), P(t|b) no-signaling (независимы от
  удалённой настройки в пределах σ). Без всех трёх — S>2 = артефакт до доказательства.
H-R4 (двойной, добавлен по итогам C2-ISO):
  (а) ФОРМА: на каждом k_f — полная θ-сетка (9 точек, шаг π/8) + фит (ρ, p̂)
      хорды E(θ)=−ρ·E_p(θ) (канон §2.6). Гипотеза: p̂ монотонно растёт с k_f.
      Регистрируемая ТРАЕКТОРИЯ (p̂, ρ, F_s, Δ, S_direct)(k_f) — ГЛАВНЫЙ выход
      кампании (не только точка излома F_s).
  (б) СТОЛБЫ: пока F_s < 3σ_F, требуется ρ_dyn ≤ 1/K_G(3) ≈ 0.66. Нарушение
      (ρ_dyn > 0.66 при факторизующейся карте) = ОБЯЗАТЕЛЬНЫЙ стоп-аудит:
      конфликт с теоремой Гротендика ⇒ по построению баг пайплайна, не физика.

## Бюджет (оценка, fp64 GPU; ОБНОВЛЁН под H-R4а полную θ-сетку)
R0 смоук: секунды. Две компоненты на ячейку (k_f×N):
 (i) F_s(k_f): сопряжённые прогоны Δγ=π/2 (первичный) × зеркала(2) = ~3 прогона;
 (ii) E(θ) шейп H-R4а: полная θ-сетка 9 точек + 2 доп. CHSH-угла = ~11 прогонов
      (как ISO-DYN, ~8 мин/ k_f-ячейка при M=4800).
4 k_f × 2 N = 8 ячеек × ~14 прогонов = ~112 прогонов M=4800. При больших k_f
lr мал ⇒ то же число шагов (6000). Оценка: ~60–90 мин (θ-шейп доминирует над
F_s-лестницей). Сокращение: E(θ)-шейп только на N=32 (κ-скейлинг F_s на N=8
достаточно точкой излома) ⇒ ~40–50 мин. РЕШЕНИЕ по объёму θ-шейпа — за Артемом
(вопрос 2/3 открыты).
