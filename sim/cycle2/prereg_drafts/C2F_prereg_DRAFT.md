# C2-F prereg DRAFT (не утверждён). Факторизуется ли релаксация.
Протокол: сопряжённые прогоны при СОВПАДАЮЩЕЙ λ — идентичные начальный
конфиг (prep_dynamics, общий key) и реализация шума (тот же PRNGKey в
степпере), различие ТОЛЬКО в удалённой настройке b vs b'. a=ê фиксирована.
Сетка b: γ ∈ {0, π/8, π/4, 3π/8, π/2} в плоскости (x,z); базовая пара
(b=ê, b'=b(γ)). Ячейки: N∈{32,96} × k_f∈{×1,×4} × T∈{0.05, 0} (T=0 —
квенч после prep). M ≥ 4800/ячейку (уточнить по бюджету GPU). Зеркальные
пары (a,b)→(−a,−b). DEGENERATE репортится, НЕ отбрасывается; sign(0)→+1.
Наблюдаемые: F(γ)=P(s(λ,a,b)≠s(λ,a,b')) [flip-rate ближнего конца];
Δ(γ)=|P(s=+|a,b)−P(s=+|a,b')| [маргинал]; заодно F для t (ближний к b).
Гипотезы архитектора (прогноз в журнал до данных):
H-F1: F(N=32,k_f×4,T=0.05) > 0, диапазон 0.02–0.10. KILL: F < 3σ_F на
  всей сетке γ при M≥4800 ⇒ факторизация эффективна, канал мёртв.
H-F2: F(96) < F(32) (затухание с длиной). KILL: незатухание (F(96) ≥ F(32)
  в пределах 2σ при подтверждённом H-F1).
H-F3: Δ совместим с 0 (χ² однородности P(s|a,γ) по γ, p>0.05). KILL
  (сюрприз-ветка): p<0.001 И Δ_max>3σ ⇒ маргинальный телеграф — стоп-аудит
  (проверка протокола до интерпретации).
Логическая связка: Δ>0 ⇒ F>0; конфигурация {F≈0, Δ>0} запрещена — её
появление = баг пайплайна по определению.
Точность: политика fp32/fp64 — по Задаче 1.6, утверждает Артем.
