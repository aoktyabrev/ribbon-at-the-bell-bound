# C2-J addendum 1: интегратор и гейт J0-v2 (решение архитектора).
Диагноз KILL: rej не масштабируется с lr ⇒ фикс-lr непригоден для
stiff-цепи; риск неконвергентности мягкой моды важнее самих rej.
Ветвление по диагностику bdj5hr2ew:
(А) ТРАНЗИЕНТ (rej в раннем окне, хвост <1e-3): burn-in фаза с
    line search/клипом шага + основная фаза прежним интегратором.
(Б) ПЕРСИСТЕНТНО: адаптивный интегратор всей траектории —
    многообразный градиентный спуск, per-replica Armijo line search
    (T=0, энергия монотонна); прежний явный путь сохраняется как
    код-путь фазы D (не трогается).
ГЕЙТ J0-v2 (заменяет rej-критерий, обязателен для обеих веток):
1. Якорь: A(k_f×1, k_c×1) = 0.369 ± 3σ_comb новым интегратором.
2. Сертификат сходимости: ≥99% реплик с max|grad| < g_tol (или
   энергетическое плато |ΔE|/шаг < e_tol на последних 5% шагов)
   В КАЖДОЙ ячейке лестницы; g_tol/e_tol фиксируются в addendum-2
   по фактическим масштабам градиента (смоук, без наблюдаемых).
3. Инвариантность бассейнов: худшая ячейка (×256), подвыборка M=256,
   половинение допуска line search ⇒ смена исходов s ≤ биномиального
   шума. Провал п.3 = стоп: бассейны не определены дискретизацией,
   решение архитектора заново.
Отчёт J0-v2 — против трёх пунктов, до лестницы.
