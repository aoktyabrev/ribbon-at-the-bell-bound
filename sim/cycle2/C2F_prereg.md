# C2-F prereg v3 (утверждён Артемом 2026-07-16). Факторизуется ли релаксация.
Точность/бэкенд: fp64 GPU (после валидации C2-V0).
Сопряжённые прогоны: идентичные prep (общий key) и сырой шумовой драйв
(общий PRNGKey степпера; b в раздачу ключей не входит — проверено,
measurement.py:260). Оговорка: побитово общим является только сырой
драйв; эффективные приращения расходятся через тангенциальную проекцию
— стандартный смысл common random numbers.
СТАДИЯ F0 — ГЕЙТ ХАОСА (обязательна, до основной кампании):
N=32, k_f×4, M=1200; F(δ) для поворота удалённой оси b на
δ ∈ {1e−6, 1e−4, 1e−2, π/8, π/2} рад; ячейки T=0.05 И T=0 (квенч: всё
расписание при T=0 после стандартного prep).
  Нуль-тест пайплайна (до лестницы): γ=0 ⇒ b'=b ⇒ F(0) обязано быть
  РОВНО 0.0 побитово. Любое F(0)≠0 — баг раздачи ключей, стоп.
  γ=0 точкой данных НЕ является.
  Ветвление F0 (критерии регистрируются до прогона; per-ячейка
  T=0.05 и T=0 раздельно):
  (i) ПРОХОД: F(1e−6) = 0 событий при M=1200, ЛИБО фит наклона
      log F(δ) по δ∈{1e−6,1e−4,1e−2} даёт slope ≥ s_min=0.5
      (утверждён Артемом 2026-07-16; теор. ожидание ~1 для
      градиентной системы с линейной мерой δ-окрестности сепаратрисы)
      И F(1e−6) ≤ 0.1·F(π/8).
      Наклон: взвешенный МНК log F(δ) vs log δ по трём точкам,
      σ_logF = σ_F/F. Точка с 0 событий в фит не входит (проход по (i));
      0 событий в ≥2 точках из трёх ⇒ проход (пол ниже разрешения).
      Ветка (i) требует ОБА условия: slope ≥ s_min И F(1e−6) ≤ 0.1·F(π/8).
  (ii) ПЛАТО (kill траекторной наблюдаемой в данной T-ячейке):
      F(1e−6) и F(1e−4) оба в пределах 2σ от F(1e−2).
      При плато в T=0.05: H-F1 живёт только в T=0.
      При плато И в T=0: траекторная F мертва целиком; C2-F
      редуцируется к дистрибутивным наблюдаемым (Δ маргинала +
      условные бины по полусфере начального n_A) — фиксируется
      как исход, не как провал.
  (iii) СЕРАЯ ЗОНА (ни проход, ни плато): стоп, решение Артема
      с записью в prereg-addendum ДО основной кампании.
ОСНОВНАЯ КАМПАНИЯ (если гейт пройден):
Первичная ячейка: T=0 (квенч). a=ê; сетка Δγ ∈ {π/8, π/4, 3π/8, π/2};
N∈{32,96} × k_f∈{×1,×4}; M≥4800/ячейку; зеркала (a,b)→(−a,−b);
DEGENERATE репортится, НЕ отбрасывается; sign(0)→+1.
Наблюдаемые считаются по образцу analysis_ds3.iso_corr (все события,
sign(0)→+1, degen — диагностикой); копирование run_ds3.corr с его
keep=~degen ЗАПРЕЩЕНО (находка г.4 мемо).
Бэкенд: fp64 GPU; backend/jax-версия/устройство пишутся в meta
сырого JSON каждого прогона.
Наблюдаемые: F_T0(Δγ) = P(s(λ,a,b)≠s(λ,a,b')) — мера сдвига бассейна;
Δ(Δγ) = |P(s=+|a,b)−P(s=+|a,b')|; симметрично F для t.
Гипотезы (прогнозы архитектора — в журнал до данных):
H-F1: F_T0(N=32,k_f×4) > 0, диапазон 0.01–0.08, монотонна по Δγ.
  KILL: F_T0 < 3σ_F на всей сетке при M≥4800 ⇒ факторизация при T=0
  конструктивно установлена; Белл применим к динамическому протоколу
  строго; трилемма — теорема для класса.
H-F2: F_T0(96) < F_T0(32). KILL: незатухание (≥ в пределах 2σ при
  подтверждённом H-F1).
H-F3: Δ совместим с 0 (χ² по Δγ, p>0.05). KILL-сюрприз: p<0.001 И
  Δ_max>3σ ⇒ маргинальный телеграф — СТОП-АУДИТ до интерпретации.
Логика: Δ>0 ⇒ F>0; конфигурация {F≈0, Δ>0} = баг пайплайна.
T=0.05-ячейки: только если F0(i) пройден; те же наблюдаемые, статус
вторичный.
