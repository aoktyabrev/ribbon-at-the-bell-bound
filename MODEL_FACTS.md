## 1. Переменные состояния и их пространства

| величина | пространство | форма | ссылка |
|---|---|---|---|
| `x_i` — узлы кривой | R⁴ ≡ ℍ, кватернионы **скаляр-первый** (w,x,y,z) | (N,4) | `band4d.py:3`, `:6` |
| `u_i` — оснащение | S³ (единичные кватернионы) | (N,4) | `band4d.py:3-4` |
| `φ_i` — SO(2)-ветка (контроль V1-C) | R (хранится **непрерывно**, unwrapped) | (N,) | `band4d.py:154-166` |

- Батч — только через `vmap` в скриптах прогонов; функции написаны на ОДНОЙ кривой
  (`band4d.py:14`).
- `u_i` — лифт SO(3)-вращения нормального трёхгранника относительно **канонического
  бишоповского репера** (`band4d.py:3-4`).
- Срез R³ = Im ℍ; «лишнее» 4-е измерение — скалярная компонента `w` = **индекс 0**.
  «Зажать w=0» = `x[..., 0] = 0` (`band4d.py:10-12`, применяется в `:269`, `measurement.py:226`).
- SO(2)-ветка: `u_i(φ_i) = (cos φ_i/2, 0, 0, sin φ_i/2)`, ось ẑ=(0,0,0,1) (`band4d.py:154-160`).
  Winding `W = (φ_{N−1} − φ_0)/2π` — телескоп, работает только потому что φ unwrapped
  (`band4d.py:163-166`).

## 2. Геометрия (входит в энергию через t и c)

- Касательные: `t_i = normalize(x_{i+1} − x_i)`, i=0..N−2; **`t_{N−1} := t_{N−2}`**
  (последний узел наследует последнее ребро, чтобы #касательных = #реперов = N)
  (`band4d.py:39-47`).
- Канонический нормальный базис: `P(t) = (t⊗i, t⊗j, t⊗k)` — правый сдвиг мнимых единиц;
  гладкий по всей S³ (параллелизуемость) (`band4d.py:50-56`, `_IMAG` в `:24-28`).
- Бишоп-транспорт: `R = H(m)·H(t_a)`, `m = normalize(t_a+t_b)` — двойное отражение
  Хаусхолдера, `H(n)v = v − 2⟨n,v⟩n` (`band4d.py:63-83`). Знаменатель подпёрт:
  `max(|t_a+t_b|, 1e-12)` (`:81`).
- Связь: `C_i = T_iᵀ P_{i+1}` (3×3, оба — ОН-базисы `t_{i+1}^⊥`), `c_i = lift(C_i)`
  (`band4d.py:114-128`). Лифт `_mat3_to_quat` берёт **только ветвь Шеппарда trace>0**
  (`:99-111`) — законно, поскольку C близка к тождеству (защита δ_tan).
- Приращения твиста: **`g_i = u_i⁻¹ ⊗ c_i ⊗ u_{i+1}`**, i=0..N−2 (`band4d.py:140-147`).
  `g_i = identity` ⇔ оснащение параллельно по Бишопу.

## 3. Члены энергии — с коэффициентами и знаками, как в коде

**Внимание: в фазе D сосуществуют ДВЕ энергии с РАЗНЫМИ зажимами.**

### 3.1 Общие члены (обе энергии)

| член | формула в коде | ссылка |
|---|---|---|
| растяжение | `E_s = k_s · Σ_{i=0}^{N−2} (|x_{i+1}−x_i| − ℓ)²` | `band4d.py:178-181` |
| изгиб | `E_b = k_b · Σ_{i=0}^{N−2} (1 − t_i·t_{i+1})` | `band4d.py:184-187` |
| твист | `E_f = k_f · Σ_{i=0}^{N−2} d_i²`, `d_i = arccos(|g_i.w|)` | `band4d.py:190-196` |

- Твист использует `geodesic(g_i, identity, spinor=False)` = **`arccos(|⟨·⟩|)`** —
  НЕспинорную («короткую») метрику (`frames.py:212-223`). Прямая цитата кода:
  «Изотропная (короткая) метрика — знак твиста несёт parity, не энергия»
  (`band4d.py:193`). То есть энергия к знаку твиста слепа **по построению**.
- Аргумент `arccos` обрезается на `GEODESIC_EPS` от ±1 ради конечного градиента
  (`frames.py:219`).
- Изгиб — НЕ квадратичный по углу: это `1 − cos`, а не `θ²`.

### 3.2 Зажим — вариант D0 (`e_total`)

    E_clamp = k_c · d²(u_end, U_target),   d = arccos(|⟨u_end, U_target⟩|),  ЗНАК: +k_c

`band4d.py:199-206`. Полная: `e_total = E_s + E_b + E_f + E_clamp(u[0],U_A) + E_clamp(u[-1],U_B)`
(`band4d.py:209-219`).

Код прямо объясняет, почему НЕспинорная метрика: зажим ±-симметричен, «аппаратура
удерживает SO(3)-ОРИЕНТАЦИЮ репера и НЕ различает лифт u/−u. Сектор (чётность) задаётся
ТОПОЛОГИЕЙ … а НЕ энергией зажима — иначе зажим тянул бы нечётный сектор раскрутиться»
(`band4d.py:202-204`).

### 3.3 Зажим — вариант измерения D1+ (`e_meas`) ← **используется во ВСЕХ научных прогонах**

    E_clamp = −k_c · (n_end · a)²,   n_end = rotmat(u_end)·ê,   ЗНАК: МИНУС

`measurement.py:39-42`. Полная: `e_meas = E_s + E_b + E_f + e_axial_clamp(u[0],a) +
e_axial_clamp(u[-1],b)` (`measurement.py:45-55`).

Два минимума `n_end = ±a` доступны по построению (`measurement.py:5`, `:40`).

**[РАСХОЖДЕНИЕ]** Это ДРУГОЙ функционал, не тот, что в `e_total`: `+k_c·arccos²` против
`−k_c·proj²`. D2/D2-ext/DS2/DS3/S1/seed-audit гоняют `build_minimizer` → `e_meas`
(`measurement.py:216`), то есть **осевой** зажим. `e_total`/`e_clamp_frame` живёт в
D0-степпере (`band4d.py:250-251`). В Model-секции нельзя писать «зажим» в единственном
числе.

### 3.4 Параметры кампаний фазы D

## 4. Шаг Ланжевена — дискретизация, шум, ренормализация

Из кода (`band4d.py:278-313`; измерительный близнец `measurement.py:231-258`):

```
gx = ∇_x E ; gu = ∇_u E                      # jax.grad от e_total / e_meas
nx = N(0,1) * sqrt(2·lr·T)                   # шум в ОБЪЕМЛЮЩЕМ R⁴, покомпонентно
nu = N(0,1) * sqrt(2·lr·T)
x_new = fix_endpoints(x − lr·gx + nx)
du    = −lr·gu + nu
du    = du − ⟨du,u⟩·u                        # проекция на касательную к S³
u_new = normalize(u + du)                    # ретракция сносом на сферу
u_new = u_new * sign(⟨u_new,u⟩ + 1e-30)      # знаковая непрерывность лифта
```

- Конвенция `x += −lr∇E + √(2·lr·T)·ξ` унаследована из фаз A–C (`band4d.py:234`).
- **Не приукрашивать:** это НЕ риманов ланжевен на S³. Шум генерится в объемлющем R⁴,
  затем шаг проецируется на касательное и сносится `normalize` — метрической поправки,
  члена Ито–Стратоновича и якобиана ретракции в коде НЕТ (`band4d.py:285-289`). Стационарная
  мера не выводилась; равновесие по T нигде не доказано.
- Дискретизация — явный Эйлер–Маруяма с шагом `lr` (он же learning rate), отдельного `dt`
  нет: `lr` играет обе роли (`band4d.py:246`).
- Температура задаётся расписанием `T_schedule` (массив длины steps), интегрируется
  `lax.scan` (`band4d.py:316-318`).
- Граничные УЗЛЫ `x_0, x_{N−1}` — жёстко переустанавливаются в `X0/XL` каждый шаг
  (`band4d.py:265-270`); граничные РЕПЕРЫ свободны, держатся только зажимом (`:235`).

## 5. Зажимы: форма и куда входят a, b

- Ось считывания: `ê = REF_AXIS = (0,0,1)` в нормальных координатах (`measurement.py:25`).
- Осевой вектор конца: `n_end = rotmat(u_end) · ê ∈ R³` (`measurement.py:33-36`).
  Зависит только от `rotmat(u_end)` ⇒ **слеп к лифту u vs −u** (SO(3)-факторизуем) — это и
  есть эталон слепоты AX (`measurement.py:35`).
- `a`, `b` входят ТОЛЬКО в зажим, `−k_c(n_A·a)²` и `−k_c(n_B·b)²` (`measurement.py:53-54`),
  и затем в правило исхода (§7). Больше нигде.
- Оси по углу: `a = (0,0,1)`, `b = (sin θ, 0, cos θ)` — поворот в плоскости (x,z)
  (`measurement.py:94-100`). Для аудита DS2 обе оси свободны:
  `a=(sin α,0,cos α)`, `b=(sin γ,0,cos γ)`, угол = |α−γ| (`measurement.py:103-108`).

## 7. Правило исхода: как из финальной конфигурации получаются s, t

`measurement.py:66-73`:

```
pa = ⟨rotmat(u_0)·ê,      a⟩
pb = ⟨rotmat(u_{N−1})·ê,  b⟩
s  = +1 если pa ≥ 0 иначе −1
t  = +1 если pb ≥ 0 иначе −1
```

- **`sign(0) → +1`** — по конвенции (`measurement.py:68`, `:71-72`). Та же конвенция
  использована при снятии детекционного лупхола в DS3.
- Исход читается ТОЛЬКО с граничных реперов `u_0`, `u_{N−1}`; интерьер в правило не входит.
- Батч: `classify_batch` (`measurement.py:76-80`).

## 8. Критерий DEGENERATE

    DECISION_EPS = 0.2
    degen = (|pa| < 0.2) | (|pb| < 0.2)

`measurement.py:63`, `:79`. Смысл по коду: «ось не выбрала знак» (`:62`).
Порог **абсолютный и жёстко зашитый**, не выведен; именно его настройко-зависимость
фабриковала CHSH=2.39 в DS3 (~36% реплик). В валидных прогонах DEGENERATE не отбрасывается
(`run_s1.py:47-49` держит `keep = ~degen`, но `degen=0` во всех точках S1/seed-audit).

## 9. Детекция сингулярностей — три независимых критерия

`band4d.py:293-312`, близнец `measurement.py:245-257`. Пороги: `DELTA_SING = 2.0e-2`,
`DELTA_TAN = 2.0e-2` (`band4d.py:31-32`), переопределяемы из конфига (`:227`).

| # | критерий | условие в коде | ссылка |
|---|---|---|---|
| 1 | временной скачок лифта | `⟨u_new, u_old⟩ < 1 − δ_sing` | `band4d.py:295-296` |
| 2 | реверс касательной (бишоп-сингулярность) | `t_i·t_{i+1} < −1 + δ_tan` | `band4d.py:300-301` |
| 3 | пространственная стена лифта | `sign(g_new.w) ≠ sign(g_old.w)` | `band4d.py:302-307` |

- Отбраковка — **по ленте целиком**: `bad = bad_u | bad_t | bad_w`, и вся конфигурация
  откатывается к предыдущей (`band4d.py:308-312`). Не по узлу, не по связи.
- Счётчик `rejected` — на ленту, накапливается по шагам (`:312`).
- В отчётах фигурирует как `rej_per_step = rej / (M · steps)` (`run_s1.py:56`).
- SO(2)-ветка вместо (1) и (3) контролирует winding: `|W − W_target| ≥ wind_tol` (0.01)
  (`band4d.py:347`).

## 10. ℤ₂-инвариант (parity)

`invariant.py:27-43`:

```
g       = u_i⁻¹ ⊗ c_i ⊗ u_{i+1}                      # (N−1,4)
g_short = g · sign(g.w + 1e-30)                      # КОРОТКИЙ лифт, w ≥ 0
L0      = normalize(u_0 · sign(⟨u_0, U_A⟩ + 1e-30))  # выравнивание к U_A
L       ← normalize(L ⊗ g_i)                         # lax.scan по цепочке
parity  = sign(⟨L, U_B⟩ + 1e-30)
```

- Это и есть «знак накопленного лифта» — величина, ради которой выбран float64.
- `+1e-30` — защита от sign(0), три раза (`:35`, `:37`, `:43`).
- Тесты T-inv-1/2/3: прямой стержень → +1; вставка 2π → −1; 4π → +1; инвариантность к 1000
  гладким деформациям (`test_invariant.py:36-41`).

## 11. Оценщик E и его σ

`run_s1.py:44-56` (идентично `run_grid_ext.py:52-64`):

```
keep    = ~degen
E       = mean over keep of (s·t)
E_mir   = mean over keep of (s_m·t_m)          # оси (−a,−b), зеркальный контроль
P_align = mean over keep of ((s·t) > 0)
σ       = sqrt( max(1 − E², 1e-9) / max(n_valid, 1) )
A       = |E|
```

- σ — **биномиальная, внутри одного батча**. Разброс МЕЖДУ прогонами она не описывает;
  измерен отдельно seed-аудитом: s_seed = 0.0235 против σ_bin = 0.0266, r = 0.88
  (`seedaudit_report.md`, коммит `a26f76b`).
- Тождество при θ=0: `A = 2·P_aligned − 1`.
- Зеркало вычисляется на ТЕХ ЖЕ репликах (`keep` общий), не независимый прогон
  (`run_s1.py:45`, `:51`).

## 12. Что НЕ надо украшать — список для Model-секции

1. **Два зажима, не один** (§3.2 vs §3.3). Все научные числа — на осевом `−k_c(n·a)²`.
2. **Ланжевен не риманов** (§4): объемлющий шум + проекция + `normalize`; стационарная мера
   не выведена, равновесие не доказано.
3. **`lr` = и шаг интегрирования, и learning rate** — отдельного `dt` в коде нет.
4. **Изгиб = `k_b(1 − cos)`**, не `k_b·θ²`.
5. **Твист слеп к знаку по построению** (`arccos|g.w|`) — это выбор, а не следствие.
6. **`t_{N−1} := t_{N−2}`** — костыль ради равенства числа касательных и реперов (§2).
7. **DEGENERATE-порог 0.2 зашит и не выведен** (§8); именно он дал детекционный лупхол.
8. **`sign(0) → +1`** — конвенция, влияет на границу (§7).
9. **Отбраковка — по ленте целиком**, не по локальному узлу (§9).
10. **σ биномиальная**, воспроизводимость — отдельная величина (§11).

## 13. [РАСХОЖДЕНИЯ] код ↔ докстроки/канон

- `measurement.py:211`: докстрока `build_minimizer` обещает
  `run(key,x0,u0,a,b,T_sched) → (x,u,rejected,parity_changes)`. Фактическая сигнатура —
  `run(key, x0, u0, X0, XL, a, b, T_schedule)`, возврат `(xf, uf, rej)` (`:230`, `:263`).
  **`parity_changes` не возвращается вообще**; `X0/XL` в докстроке пропущены.
- `measurement.py:9`: «В D1 разрешены только θ∈{0,π}» — актуально для D1; D2+ гоняет
  произвольные θ через `apparatus_axes_theta` (`:94`).
- §3.3 против канона: канон описывает зажим как осевой `−k_c(n_end·a)²` — совпадает;
  но `band4d.e_clamp_frame` (D0) канон не отражает.
