# SPEC: ribbon-sim — измерение эмпирического p упругой ленты

**Проект:** quantum_entanglement / sim
**Роль этого документа:** единственный источник истины для Claude Code.
**Контекст:** см. `ribbon_model_note.md` в корне репозитория (модель, результаты, открытая задача §3).

---

## 1. Научный вопрос

Модель общей ленты воспроизводит квантовые корреляции синглета, если вероятность
ветви (s,t) пропорциональна квадрату хорды: P(s,t) ∝ |s·a − t·b|². Открытая задача —
получить этот квадрат как меру бассейнов релаксации реального двумерного объекта.

**Вопрос эксперимента:** если взять классическую дискретную упругую ленту (цепочка
ориентационных фреймов на SU(2)), зажать концы потенциалами с осями a и b, и
релаксировать из случайных микросостояний — по какому закону распределятся ветви?

Измеряем эмпирическую кривую E(θ) и фитим против семейств:
- **пила**: E = 2θ/π − 1 (детерминированные конусные бассейны, классика)
- **семейство хорды**: E_p(θ) = (sin^p(θ/2) − cos^p(θ/2)) / (sin^p(θ/2) + cos^p(θ/2))

Ожидание по умолчанию (заявляем честно ДО запуска): пила или p ≈ 1.
p = 2 стало бы крупным результатом и потребует параноидальной проверки.
Любой исход информативен: расхождение с p=2 укажет, какой структуры не хватает ленте.

## 2. Физическая модель

### 2.1 Состояние
Лента = цепочка из N ориентационных фреймов, каждый — единичный кватернион:
`q[i] ∈ S³, i = 0..N−1`. Батч из B независимых лент: тензор `(B, N, 4)`.

Ось конца: `n(q) = R(q) · e_z` (третий столбец матрицы поворота).
`n_A = n(q[0])`, `n_B = n(q[N−1])`.

### 2.2 Энергия
```
E_total = E_elastic + E_clamp_A + E_clamp_B

E_elastic  = k_e * Σ_{i=0}^{N-2} d(q[i], q[i+1])²          # изотропная фаза 1
d(p, q)    = arccos(|<p, q>|)                                # геодезия на S³/±
E_clamp_A  = −k_c * (n_A · a)²                               # два минимума: n = ±a
E_clamp_B  = −k_c * (n_B · b)²
```
Квадратичный зажим ±-симметричен: никакого встроенного предпочтения знака.
ВАЖНО: |<p,q>| в d() — осознанный выбор фазы 1 (лента без различия q/−q, т.е.
нить). Фаза 3 убирает модуль и отличает q от −q (истинная лента, 720°).

### 2.3 Динамика
Ланжевеновская релаксация с отжигом, силы через autodiff:
```
grad   = jax.grad(E_total)(q)
q     += −lr * grad + sqrt(2 * lr * T) * normal_noise
q      = normalize(q, axis=-1)                # проекция обратно на S³
T      = T0 * decay^step                      # геометрический отжиг до T≈0
```
Финал: `s = sign(n_A · a)`, `t = sign(n_B · b)`.

### 2.4 Начальные условия
Фаза 1: каждый q[i] — независимый Haar-равномерный кватернион (мера объявлена
заранее: равномерная, никакой подгонки плотности — см. note §2.7, «закон
сохранения постулата»).

## 3. Пре-регистрированные эксперименты

Каждый эксперимент = конфиг в `experiments/RX.yaml` + результат в `results/RX/`.
Предсказания фиксируются в конфиге ДО первого запуска.

**R0 — контроль «разрезанная лента».** k_e = 0 (связи нет). Обязано быть:
E(θ) ≈ 0 для всех θ, маргиналы 0.5. Если нет — баг, стоп.

**R1 — базовый.** Изотропная цепочка, T=0 (чистый градиентный спуск от Haar-случайных
начальных фреймов). Свип θ = 0..180° шагом 7.5°, ≥ 20 000 релаксаций на точку.
Вопрос: пила, p-семейство или что-то третье?

**R2 — термические бассейны.** Ланжевен с медленным отжигом (3+ скорости отжига).
Вопрос: меняет ли температурная мера форму кривой относительно R1?

**R3 — анизотропия.** Разделить E_elastic на изгиб и скрутку (дискретный стержень
Коссера: компоненты лог-отображения относительного кватерниона вдоль/поперёк оси),
свип отношения k_twist/k_bend ∈ {0.1, 1, 10}. Вопрос: двигает ли скрутка p?

**R4 — нулевая суммарная скрутка.** Начальные условия с ограничением
Π relative_rotations = identity (против свободных). Вопрос: роль закона сохранения.

**R5 — спинорная лента.** Убрать модуль в d(p,q): различать q и −q
(720°-периодичность, трюк с ремнём). Это главный кандидат на сдвиг к p=2.
Прогнать R1+R2 в спинорном режиме.

Критерии остановки фазы: R0 прошёл + R1 дал устойчивую кривую (bootstrap CI
не пересекает более одной модели-соперника).

## 4. Протокол анализа (фиксирован до запусков)

1. На каждую точку θ: счётчики ветвей (n_pp, n_pm, n_mp, n_mm).
2. Обязательные контроли на КАЖДОМ прогоне:
   - маргиналы P(s=+), P(t=+) ∈ 0.5 ± 3σ (внутренний no-signaling);
   - симметрия P(s,t) = P(−s,−t) в пределах шума;
   - воспроизводимость на 2 сидах.
3. Фит MLE по счётчикам ветвей для трёх моделей: пила, хорда-p (p свободный),
   хорда-p=2 (фикс). Сравнение по AIC. Bootstrap CI на p (1000 ресемплов).
4. Гармонический анализ E(θ): амплитуды cos θ и cos 3θ (связь с ε-семейством,
   note §2.4).
5. Отчёт эксперимента: `results/RX/report.md` — конфиг, кривая, фиты, контроли,
   вердикт против пре-регистрированного предсказания.

## 5. Технический стек

- Python 3.11+, **JAX + CUDA** (WSL2, RTX 4070 Ti 12GB). Проверка: `jax.devices()`
  должен показать gpu. Fallback CPU допустим для тестов, не для прогонов.
- Вся динамика — `jax.vmap` по батчу + `jax.lax.scan` по шагам, jit-компиляция.
  Ориентир: B = 4096 лент параллельно, N = 64 фрейма, 2000–5000 шагов.
  Память: (4096, 64, 4) float32 ≈ 4 MB — влезает с огромным запасом; батч можно
  поднимать до 32k.
- float32; сиды через `jax.random.PRNGKey`, полный детерминизм конфига.
- Визуализация: matplotlib (кривые E(θ), фиты, гистограммы ветвей) +
  экспорт 3D-снапшотов релаксации одной ленты (plotly html) для глаза.
- Без фреймворков поверх: чистый JAX + numpy + matplotlib + pyyaml.

## 7. Валидация (пишется ДО экспериментов)

- `test_frames.py`: normalize, композиция поворотов, n(identity)=e_z,
  геодезия d(q,q)=0, d(q,−q)=0 в фазе 1 и =π в спинорном режиме.
- `test_energy.py`: минимумы зажима ровно при n = ±a; градиент конечными
  разностями vs jax.grad (rtol 1e-4).
- `test_dynamics.py`: энергия монотонно не растёт при T=0; норма кватернионов
  сохраняется; батч-независимость (лента j не влияет на ленту k).
- `test_analysis.py`: MLE на синтетических счётчиках, сгенерированных при
  известном p, восстанавливает p ± CI; пила распознаётся против p-семейства.

## 8. Definition of Done по фазам

- **Фаза A (каркас):** структура, frames/energy/dynamics, все тесты зелёные,
  R0 прошёл контроль.
- **Фаза B (базовая наука):** R1 + R2 прогнаны на 2 сидах, отчёты с фитами и CI.
- **Фаза C (структурные вариации):** R3, R4, R5, сводный отчёт: таблица
  «структура ленты → эмпирическое p».
- **Фаза D (по результатам):** обсуждение с архитектором, дизайн следующих
  экспериментов.

## 9. Правила против самообмана

1. Пре-регистрированное предсказание в yaml не редактируется после запуска.
2. Любое «о, получилось p≈2» → немедленно: второй сид, удвоение N и B,
   проверка контролей, поиск бага в классификации ветвей. Экстраординарные
   результаты требуют экстраординарной паранойи.
3. Никогда не подгонять параметры модели (k_e, k_c, отжиг) «чтобы кривая была
   красивее» без записи в лог решений (`results/decisions.md`).
4. Отрицательный результат оформляется с тем же качеством, что положительный.
