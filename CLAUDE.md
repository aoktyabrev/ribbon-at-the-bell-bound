# CLAUDE.md — ribbon-sim

## Роль
Ты — исполнитель вычислительного физического эксперимента. Архитектура и научный
дизайн зафиксированы в `SPEC.md` — он главнее любых догадок. Научный контекст —
`ribbon_model_note.md` (в корне репозитория, рядом с этим файлом; код — в `sim/`).
Архитектурные решения не менять без пометки
`ARCH-QUESTION:` в ответе и остановки.

## Окружение
- WSL2 Ubuntu, RTX 4070 Ti (12GB), CUDA. JAX обязан видеть GPU: перед первым
  прогоном выполни и покажи `python -c "import jax; print(jax.devices())"`.
- Виртуальное окружение uv или venv в `sim/.venv`.
- Прогоны R1+ занимают минуты — запускай сначала smoke-версию (B=256, 200 шагов,
  3 точки θ), только затем полную.

## Рабочий цикл
1. Фаза A целиком (каркас + тесты + R0) до любых научных прогонов.
2. Каждый эксперимент: smoke → полный прогон → контроли §4.2 SPEC → report.md.
3. Коммит после каждой зелёной фазы; сообщение вида `phase-A: frames+energy+R0 green`.
4. Если контроль §4.2 провален — стоп, диагноз, не продолжать науку на битом коде.

## Стиль кода
- Чистый JAX: jit + vmap + lax.scan; никаких питоновских циклов по шагам/батчу.
- Функции чистые, состояние — явные пайтри. Комментарии по-русски, идентификаторы
  по-английски.
- Каждая физическая формула в коде — со ссылкой на раздел SPEC (`# SPEC §2.2`).
- float32, PRNGKey из конфига, никакого глобального random.

## Категорически
- Не редактировать пре-регистрированные предсказания в experiments/*.yaml после запуска.
- Не подгонять физические параметры ради красоты кривой без записи в results/decisions.md.
- Не интерпретировать результаты за пределами протокола §4 — интерпретация
  делается с архитектором.
- При p, близком к 2: включить протокол паранойи (SPEC §9.2) до любых выводов.
